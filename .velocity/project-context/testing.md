# Testing Standards — Velocity Platform

> Not generated by /init — hand-authored for the Velocity development repository.
> The Velocity platform produces prompt artifacts (SKILL.md, YAML, Markdown), not code.
> Velocity tests itself through RALPH runs: execute a skill → annotate the output → extract patterns.
> Referenced by: tdd skill (if applied to Velocity internals), QA Agent, Engineer Agent context load.

---

## Philosophy

- Skills are tested by running them: execute the skill on a representative input, observe the output, annotate.
- Agent configs are tested by running Agent Factory and inspecting the generated `.velocity/agents/` output.
- Schemas are tested by validating representative YAML/JSON against the schema.
- RALPH annotations are the primary quality signal — not unit tests.

---

## What Counts as a Test for Velocity Artifacts

| Artifact type   | Test mechanism                                                      |
| --------------- | ------------------------------------------------------------------- |
| `SKILL.md`      | RALPH run: execute on a sample repo, annotate output quality        |
| `agent.md`      | Schema validation + Agent Factory dry-run output review             |
| `guardrails.md` | Schema validation + validate skill dry-run on a branch              |
| Template files  | Manual inspection: does `/init` produce the right output on a repo? |
| Schema files    | JSON Schema validation with representative valid + invalid samples  |

---

## Coverage Requirements

- Every new skill must be exercised in at least one RALPH annotated run before being marked complete.
- Every schema change must be validated with at least one valid and one invalid sample.
- Agent Factory changes must produce a dry-run output reviewed by the Reviewer Agent.

---

## TDD Loop (when applicable)

If implementing a feature that produces a skill or agent config as output:

```
1. Define the expected output (what should the generated skill look like?)
2. Write the skill template (red: does it produce the right output on a test repo?)
3. Run the skill (green: output matches expected)
4. Refactor the skill prompt for minimalism and deep module alignment
5. Annotate with RALPH
```

---

## Feedback Loop Gates (for Velocity development)

- **Typecheck:** Schema validation — run `ajv validate` or equivalent against all modified schemas
- **Test:** Execute the modified skill on the Velocity test repository; inspect output
- **Lint:** Spell-check and CONTEXT.md term consistency in modified skill/agent files

---

## Stack-Specific Notes

The Velocity platform has no runtime test framework (Jest, JUnit, pytest, etc.).
The test surface is RALPH: structured annotation of agent skill runs.
See `.velocity/artifacts/ralph/` for accumulated run annotations.
