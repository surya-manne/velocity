# .velocity/ Structure Reference

Complete reference for the `.velocity/` directory — Velocity's canonical configuration store.

## Full Structure

```
.velocity/
│
├── project-intelligence/
│   ├── stack.md              ← 12-dimension stack fingerprint (generated by /init and /sync)
│   └── stack-delta.md        ← Changed fields only (generated by /sync, transient)
│
├── project-context/
│   ├── engineering.md        ← Coding standards, patterns, architecture decisions
│   ├── testing.md            ← Test strategy, coverage thresholds, test tooling
│   ├── security.md           ← Security requirements, auth/authz standards
│   └── api.md                ← API design conventions, versioning, error formats
│
├── context/
│   ├── CONTEXT.md            ← Shared cross-context ubiquitous language
│   ├── orders/
│   │   └── CONTEXT.md        ← Orders bounded context glossary
│   ├── payments/
│   │   └── CONTEXT.md        ← Payments bounded context glossary
│   └── [context]/
│       └── CONTEXT.md        ← Per-context glossary (one per bounded context)
│
├── context-map.md            ← Bounded context index and relationships
│
├── guardrails/
│   ├── default.md            ← Primary guardrail and loop configuration (YAML)
│   └── packs.md              ← Guardrails imported from rule packs
│
├── agents/
│   ├── engineer.md           ← Engineer agent configuration
│   ├── architect.md          ← Architect agent configuration
│   ├── product.md            ← Product agent configuration
│   └── [agent].md            ← One file per agent role
│
├── skills/
│   ├── tdd.md                ← TDD skill with stack-specific commands
│   ├── validate.md           ← Validate skill with threshold configuration
│   └── [skill].md            ← Configured skills (generated by skill-factory)
│
├── knowledge-base/
│   ├── index.md              ← Knowledge base index
│   ├── adrs/
│   │   ├── index.md          ← ADR index
│   │   ├── ADR-0001.md       ← Architecture Decision Records
│   │   └── ADR-[n].md
│   ├── incidents/
│   │   ├── index.md
│   │   └── INC-[n].md        ← Post-mortems and incident reports
│   ├── runbooks/
│   │   ├── index.md
│   │   └── [runbook].md      ← Operational runbooks
│   ├── git-digest/
│   │   └── index.md          ← Summarized commit history
│   └── product/
│       └── index.md          ← Product decisions and research
│
├── rule-packs/
│   ├── imported.md           ← Index of all imported packs
│   ├── skills/
│   │   └── [skill].md        ← Skills from rule packs
│   └── guardrails/
│       └── packs.md          ← Guardrails from rule packs
│
└── artifacts/
    ├── prd/
    │   └── [feature].md      ← Product Requirement Documents (from /to-prd)
    ├── features/
    │   └── [feature].md      ← Vertical slice boards (from /to-features)
    ├── tasks/
    │   └── [feature].md      ← Task queues with blocking graph (from /to-tasks)
    ├── handoffs/
    │   ├── index.md          ← Handoff index
    │   └── [task-id].md      ← Task/slice handoff artifacts (from /handoff)
    ├── audit/
    │   └── audit.jsonl       ← Append-only audit log (JSON Lines)
    ├── approvals/
    │   └── index.md          ← Approval records index
    ├── context-proposals/
    │   └── [session].md      ← CONTEXT.md proposals (from /grill-with-docs)
    └── loop/
        ├── state.md          ← Loop execution state (from /loop)
        └── [task-id]-result.md ← Task results from loop runs
```

## File Ownership

| File/Directory                | Written By                                        | Read By                   |
| ----------------------------- | ------------------------------------------------- | ------------------------- |
| `stack.md`                    | `/project-intelligence`                           | All agents, all adapters  |
| `project-context/`            | Templates + manual                                | All agents                |
| `context/CONTEXT.md`          | `/grill-me`, `/grill-with-docs`, `/context-merge` | All agents                |
| `context-map.md`              | `/init`, `/sync`                                  | Architect, Planner        |
| `guardrails/default.md`       | `/guardrail-factory`                              | All adapters, `/validate` |
| `agents/*.md`                 | `/agent-factory`                                  | All adapters              |
| `knowledge-base/`             | `/ingest`, `/adr-engine`                          | All agents                |
| `artifacts/prd/`              | `/to-prd`                                         | `/to-features`            |
| `artifacts/features/`         | `/to-features`                                    | `/to-tasks`               |
| `artifacts/tasks/`            | `/to-tasks`                                       | `/tdd`, `/loop`           |
| `artifacts/audit/audit.jsonl` | All skills                                        | `/audit-trail`            |

## Never Modified By Velocity

These files are only written by humans:

- `project-context/*.md` — After initial template, these are maintained manually
- `context/CONTEXT.md` — After initial scaffold; proposals via `/grill-with-docs`; merges via `/context-merge`
- `knowledge-base/adrs/` — ADRs are proposed by `/adr-engine` but approved/committed by humans
- `artifacts/approvals/` — Approval records require human sign-off

## Committing .velocity/

All of `.velocity/` should be committed to the repository except:

```gitignore
# Transient files — don't commit
.velocity/project-intelligence/stack-delta.md
.velocity/artifacts/loop/
```

The rest of `.velocity/` is the configuration source-of-truth. Commit it, review it in PRs, and treat it as production code.
