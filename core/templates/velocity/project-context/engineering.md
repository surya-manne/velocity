# Engineering Standards

> Generated by Velocity /init. Customize for your team.
> Referenced by: Engineer Agent, Architecture Agent, all skill context loads.

## Architecture

- Vertical slice by default. Every feature delivery is end-to-end.
- Tracer bullet first: build the thinnest possible path before expanding.
- Start a fresh context window for each new slice.
- Deep modules: prefer complex implementations with simple interfaces over many small files.
- No horizontal layers: do not build all DB schema, then all APIs, then all UI.

## Code Standards

- Variable names, file names, and API terms must match CONTEXT.md glossary exactly.
- No orphaned code: every function, class, or module must have at least one caller.
- Fail fast: validate inputs at system boundaries; do not pass invalid state inward.

## Dependencies

- Prefer established, widely-used libraries over niche alternatives.
- Pin dependency versions in production.
- Audit new dependencies for security vulnerabilities before adding.

## Error Handling

- All errors must be handled explicitly. No swallowed exceptions.
- Error messages must be actionable: say what failed and how to fix it.
- Log errors at the boundary where they cross system layers.

## Commits and PRs

- Each PR must be a complete vertical slice (UI + API + persistence + tests).
- PR description must include: what was built, how to test it, and the acceptance criteria it satisfies.
- No direct commits to `main` or `master`. All work via branches.

## Review Standards

- Every PR requires at least one human review.
- Reviewers check for: slice completeness, CONTEXT.md term consistency, test coverage, and no horizontal layering.
